One remark regarding Tor and being discoverable on the Lightning Network as I think is still not completely clear:

Tor works on the Internet (TCP/IP) layer. The connections on the Internet are between IP addresses (masked to be domains or URL-s). Tor obfuscates the origin and destination of packages with a layered encryption tech called onion routing (hence the logo).

LN is a network in itself layered on top of the Internet but not dependent on it. The communication is between URI-s (Unified Resource Identifiers = LN node public keys).

The confusion might have been caused by the fact that LN routing uses the underlying principle of onion routing too (layered encrypted packages) for the communication between nodes.

In the roundtable you spoke about that regardless of running a node behind Tor a node gets publicly advertised on the Lightning Network once it has a public channel. That info becomes the part of the public gossip about routes.

That's why there can be a routing node running behind Tor. The LN channel info is public, only it does not get tied to an IP address.

Hope that clears it a bit.
